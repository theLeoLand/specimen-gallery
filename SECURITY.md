# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Specimen Gallery, please report it responsibly.

**Do NOT open a public issue for security vulnerabilities.**

Instead, please email: **specimengallery.helpdesk@gmail.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

## What to Expect

- We will acknowledge your report within 48 hours
- We will investigate and keep you informed of progress
- We will credit you in the fix (unless you prefer to remain anonymous)

## Scope

This policy applies to:
- The Specimen Gallery application code
- The official deployment at specimen.gallery

Out of scope:
- Third-party services (Cloudinary, Fly.io, etc.) — report to them directly
- Social engineering attacks
- Denial of service attacks

## Supported Versions

We only support the latest version deployed to production.

## Security Best Practices for Contributors

- Never commit secrets, API keys, or credentials
- Use environment variables for sensitive configuration
- Keep dependencies updated
- Follow Rails security best practices

Thank you for helping keep Specimen Gallery secure.
